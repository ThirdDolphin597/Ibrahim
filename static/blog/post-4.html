#include <Adafruit_NeoPixel.h>
#include <avr/power.h>

#define NEO_PIN 6
int analogPin = A0;
#define NUMPIXELS 8
#define PRESSED LOW

Adafruit_NeoPixel pixels = Adafruit_NeoPixel(NUMPIXELS, NEO_PIN, NEO_GRB + NEO_KHZ800);

void setup() {
  pixels.begin();
  pinMode(A0,INPUT);
  Serial.begin(9600);
}

const uint8_t edge_colors[NUMPIXELS][3] = {
  { 85, 213, 85 },
  { 43, 195, 210 },
  { 64, 191, 255 },
  { 64, 191, 255 },
  { 5, 127, 215 },
  { 5, 127, 215 },
  { 16, 78, 146 },
  { 16, 78, 146 }
  
};

void set_edge_colors(int deelay) {
  for (int led_idx = 0; led_idx < NUMPIXELS; led_idx++) {
    pixels.setPixelColor(led_idx, pixels.Color(edge_colors[led_idx][0], edge_colors[led_idx][1], edge_colors[led_idx][2]));
    delay(deelay);
    pixels.show();
  }

}

void set_all_leds(int r, int g, int b) {
  for (int i = 0; i < NUMPIXELS; i++) {
    pixels.setPixelColor(i, pixels.Color(r, g, b));
  }
  pixels.show();
}



void loop() {
  int buttonState = analogRead(analogPin);
   Serial.println(buttonState);
  if (buttonState < 0.2) {
    set_all_leds(0, 0, 0);
    delay(250);
    set_all_leds(255, 3, 0);
    delay(750);
    set_all_leds(0, 0, 0);
    delay(250);
    set_all_leds(255, 3, 0);
    delay(250);
    set_all_leds(0, 0, 0);
    set_edge_colors(100);

  } else {
    set_edge_colors(0);

  }
    pixels.show();
}

Code Explanation

Switching to the Hall Effect Sensor

#define NEO_PIN 6
int analogPin = A0;
#define NUMPIXELS 8
#define PRESSED LOW

The biggest change from the last version is ditching the button entirely. Instead of #define BUTTON_PIN 12, I'm now using int analogPin = A0; to hook up the hall effect sensor. Since the sensor outputs an analog voltage that changes based on magnetic field strength, I need to read from an analog pin instead of just getting a simple HIGH/LOW from a digital pin.

Adding Serial Communication

void setup() {
  pixels.begin();
  pinMode(A0,INPUT);
  Serial.begin(9600);
}

I added Serial.begin(9600) so I can see what the sensor is actually reading in the Serial Monitor. This is important because I need to figure out what threshold value to use to detect when the magnet is close enough. Without being able to upload to my own board, this was going to be tricky—more on that later.

Making the LEDs Actually Update

void set_edge_colors(int deelay) {
  for (int led_idx = 0; led_idx < NUMPIXELS; led_idx++) {
    pixels.setPixelColor(led_idx, pixels.Color(edge_colors[led_idx][0], edge_colors[led_idx][1], edge_colors[led_idx][2]));
    delay(deelay);
    pixels.show();
  }
}

I also added pixels.show() inside the loop this time. Before, I was only calling it once at the very end, which meant the cascading animation wouldn't actually show each LED lighting up one by one. Now with pixels.show() called inside the loop, each LED updates immediately, giving you that nice waterfall effect.

void set_all_leds(int r, int g, int b) {
  for (int i = 0; i < NUMPIXELS; i++) {
    pixels.setPixelColor(i, pixels.Color(r, g, b));
  }
  pixels.show();
}

Same thing here—I added the pixels.show() call so the color changes actually display on the strip immediately.

Reading the Sensor

int buttonState = analogRead(analogPin);
Serial.println(buttonState);

Instead of digitalRead(BUTTON_PIN) which gives me 0 or 1, I'm now using analogRead(analogPin) to get values from 0-1023. The Serial output prints the sensor value so I can watch it change as I move a magnet around.

The Threshold Check

if (buttonState < 0.2) {

Instead of if (buttonState == PRESSED), I'm now comparing the analog value to a threshold. When the magnetic field is strong enough, the sensor value drops below (x) and triggers the LED sequence. The exact threshold value needs to be tuned based on how sensitive I want the activation to be.

The Upload Problem

Here's where things got annoying. When I tried to upload this sketch to my microcontroller, I got this error:

avrdude: ser_open(): can't open device "\\.\COM3": Access is denied

Basically, Windows wouldn't let the Arduino IDE access COM3. I'm not sure if something else is holding onto that port or if there's a driver issue, but it was blocking me from uploading. Since I need to see the actual sensor readings to calibrate the threshold, I had to get creative. My solution was to email the code to a friend with a working Arduino setup and have them upload it for me from their device. It's not ideal—it's slow and makes iteration harder—but it's the only way I could get the sensor data I need right now.
